# BIELSYS Tools

Dette repositoriet inneholder ulike verktøy som kan være nyttige gjennom BIELSYS-årene. Målet er å samle små og store hjelpemidler som kan forenkle arbeid, testing, læring og utvikling innen faget.

## Bidrag

Bidrag er hjertelig velkomne!

For større endringer, vennligst opprett en issue først slik at vi kan diskutere forslaget før du begynner arbeidet.

### Retningslinjer for utvikling
- `main` er beskyttet — vennligst opprett en egen branch når du utvikler nye funksjoner eller fikser feil.
- Opprett en pull request når du er klar for å få endringene inn i `main`.
- Husk å oppdatere tester og dokumentasjon ved behov.

### Standard bidragsprosess

1. Fork repositoriet: Opprett din egen kopi av dette repositoriet under din konto
2. Klon repositoriet: Klon det forkede repositoryet til din lokale maskin (eller hvor enn du vil jobbe med koden).
3. Utfør endringer: Gjør ønskede endringer og opprett en commit.
4. Push til din fork: Push commits fra din lokale maskin til det forkede repositoriet på GitHub.
5. Opprett en pull request: Åpne en pull request fra din fork til det opprinnelige repoet. Dette er en formell forespørsel til eieren av det opprinnelige repoet om å gjennomgå og godta endringene dine.
6. Eier gjennomgår og slår sammen: Eier av det opprinnelige repositoriet kan deretter gjennomgå pull requesten din og, hvis de godkjenner, slå sammen endringene dine med hovedprosjektet.
