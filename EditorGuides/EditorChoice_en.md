# Choice of editor/IDE

Choosing an editor or IDE is a personal preference, and there are many good options out there. Here are some suitable choices for BIELSYS students that we will look at: Visual Studio Code, Arduino IDE, and CLion. In this we will take a quick look at what the different options have to offer, but more information will be available in the dedicated folders.

## Visual Studio Code
Visual Studio Code (VS Code) is a popular, free, and open-source editor developed by Microsoft. It is lightweight, fast, and has a large ecosystem of extensions that make it highly customizable. VS Code supports many programming languages and has built-in features for debugging, Git integration, and more. It is also often used in other courses such as Prosedyre- og objektorientert programmering (TDT4102) and Programmering, numerikk og sikkerhet (INGT1002). In this program there is quite a bit of programming of microcontrollers such as Arduinos and AVRs. Then VS Code can be used together with extensions for Arduino (PlatformIO) and AVR (MPLAB or PlatformIO). MPLAB has an official IDE that can be used, but it also has an extension for VS Code. Even though it is possible to program AVRs using PlatformIO, we will not look at that here since it adds more obscurity than the purpose of the instruction.

## Arduino IDE
Arduino IDE is a simple and intuitive editor specifically designed for Arduino microcontrollers. It is free and available for multiple platforms. Arduino IDE has built-in features for uploading code to Arduino devices and supports many different Arduino platforms. It is especially useful for beginners in microcontrollers and electronics. It is made to be very easy to learn and use, and is therefore a good choice for those who are new to programming or microcontrollers. It is also a good introduction to more advanced IDEs such as Visual Studio Code and CLion later in the program. That said, it is recommended to use Visual Studio Code for most tasks in BIELSYS, as it provides more flexibility and more features that can be useful in the long run. Arduino IDE, among other things, will not handle the same complexity when it comes to debugging and testing, and it is therefore not recommended for more advanced projects. It also "helps" by hiding a lot of the complexity in the background, which can be a disadvantage when you want to learn more about how things work under the hood or if you want to look more at how the different libraries are written.

## CLion
CLion is a professional IDE developed by JetBrains, specifically designed for C and C++ programming. It is a powerful IDE with many advanced features such as intelligent code completion, refactoring, and built-in debugging. CLion also supports CMake, which is a popular build system for C++ projects. It is a good choice for more experienced programmers who want a robust and feature-rich IDE for C++ development. CLion does not support MPLAB (at the time of this writing) and it is therefore not recommended for AVR programming, but it does have support for Arduino through the PlatformIO plugin. It is also worth noting that CLion is a commercial IDE and requires a license for full functionality or if you make money from what is developed (read their license terms for more information). However, it is free for students and teachers through JetBrains' education program, so it can be a good alternative for those who qualify. CLion can be overkill for simpler projects or for those who are new to programming, but it can be an excellent choice for more complex projects or for those who want to learn more about advanced C++ features and tools. It can also help provide a good introduction to code structure and organization since it has tools for enforcing code style and organization. It is also worth noting that CLion has a number of built-in tools for testing and debugging, which can be very useful for more complex projects. It can also be a good choice for those who want to learn more about professional software development and tools used in industry.

## Summary
The choice of editor or IDE depends on your personal preferences, experience, and needs. For beginners, Arduino IDE can be a good choice to get started with microcontroller programming, while Visual Studio Code and CLion offer more advanced features for those who want to learn more and work with more complex projects. It is recommended to try out different editors and IDEs to find the one that suits you best and your needs in the BIELSYS program. Remember that the most important thing is to find a tool you are comfortable with and that supports your learning goals and projects effectively. Regardless of which editor or IDE you choose, it is important to invest time in learning how to use it effectively, as this can have a big impact on your productivity and learning experience throughout the BIELSYS years.

The author's recommendation is to use Visual Studio Code for most students at BIELSYS (but it is not guaranteed that student assistants can help and it will not be taught), as it provides a good balance between ease of use and advanced features.

## Quick note on using AI in editors
Many modern editors and IDEs now have integrated support for artificial intelligence (AI) and machine learning (ML) to improve code completion, debugging, and the overall development experience. For example, Visual Studio Code has extensions such as GitHub Copilot, which uses AI to suggest code and help with autocompletion based on the context of what you are writing. CLion also supports AI-based tools for code analysis and refactoring. These tools can in some cases be helpful, but there is some debate about how useful they actually are. Among other things, a study published in 2025 indicated that AI (including GPT-4o and Claude 3.5) cannot fix more than a few simple tasks (https://futurism.com/openai-researchers-coding-fail). It is also worth noting that AI tools can have limitations and may not always understand the context or the intention behind your code, which can lead to errors or inefficient suggestions. It can also contribute to reduced understanding of "your" code. If you still choose to use AI, there are some recommendations that should be considered.

Consider turning off "inline suggestion": "Inline suggestion" only gives you a suggestion for what to do next, it does not have any more context than what you have written so far. In some cases it turns out that it is wrong, which can take more time to fix than you would have used to solve the problem with other methods. It is also much easier to lose track of what the code actually does compared to how it operates in a chat. In a chat it will at least try to explain what it has done and "thought". Chat can therefore be a better alternative than inline suggestions, but still offers some challenges, as mentioned AI has problems with complex tasks. There are also some environmental considerations that should be made before choosing to use AI, but we will not look at that here (video on AI resource use: https://youtu.be/H_c6MWk7PQc?si=s-Rn35bYn43XBtoY).
