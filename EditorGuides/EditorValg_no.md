# Valg av editor/IDE

Valg av editor eller IDE er en personlig preferanse, og det finnes mange gode alternativer der ute. Her er noen av de passende valgene for BIELSYS-studenter som vi skal se på: Visual Studio Code, Arduino IDE, og CLion. I denne kommer vi til å ta en kjapp tit på hva de forskjellige alternativene har å tilby, men mere informasjon vil være tilgjengelig i de dedikerte mappene.

## Visual Studio Code
Visual Studio Code (VS Code) er en populær, gratis og åpen kildekode editor utviklet av Microsoft. Den er lettvektig, rask og har et stort økosystem av utvidelser som gjør den svært tilpassbar. VS Code støtter mange programmeringsspråk og har innebygde funksjoner for debugging, Git-integrasjon, og mer. Det er også ofte brukt i andre fag som Prosedyre- og objektorientert programmering (TDT4102) og Programmering, numerikk og sikkerhet (INGT1002). Dette studiet så gjøres det en del programmering av mikrokontrollere som Arduinoer og AVRer. Da kan VS Code brukes sammen med utvidelsene for Arduino (PlatformIO) og AVR (MPLAB eller PlatformIO). MPLAB har en offisiell IDE som kan brukes, men den har også en extension for VS Code. Selv om det er mulig å programmere AVRer med hjelp av PlatformIO så vil vi ikke se på det her siden det gir er større obskuritet en det som er formålet med undervisningen.

## Arduino IDE
Arduino IDE er en enkel og intuitiv editor spesielt designet for Arduino-mikrokontrollere. Den er gratis og tilgjengelig for flere plattformer. Arduino IDE har innebygde funksjoner for å laste opp kode til Arduino-enheter, og støtter mange ulike Arduino-plattformer. Den er spesielt nyttig for nybegynnere i mikrokontroller og elektronikk. Den er laget for å være veldig enkel å sette seg inn i og bruke, og er derfor et godt valg for de som er nye til programmering eller mikrokontrollere. Det er også en god introduksjon til mer avanserte IDE-er som Visual Studio Code og CLion senere i studiet. Det sagt så anbefales det å bruke Visual Studio Code for de fleste oppgaver i BIELSYS, da det gir mer fleksibilitet og flere funksjoner som kan være nyttige i lengden. Arduino IDE vil blant annet ikke støte samme kompleksiteten når det kommer til debugging og testing, og det er derfor ikke anbefalt for mer avanserte prosjekter. Den "hjelper" også med å skjule mye av kompleksiteten i bakgrunnen, noe som kan være en ulempe når man ønsker å lære mer om hvordan ting fungerer under panseret eller om man ønsker å se mere på hvordan de forskjellige bibliotekene er skrevet.

## CLion
CLion er en profesjonell IDE utviklet av JetBrains, spesielt designet for C og C++ programmering. Det er en kraftig IDE med mange avanserte funksjoner som intelligent kodefullføring, refakturering, og innebygd debugging. CLion støtter også CMake, som er et populært byggesystem for C++ prosjekter. Det er et godt valg for mer erfarne programmerere som ønsker en robust og funksjonsrik IDE for C++ utvikling. CLion støter ikke MPLAB (når denne teksten skrives) og det er derfor ikke anbefalt for AVR-programmering, men den har støtte for Arduino gjennom PlatformIO pluginen. Det er også verdt å merke seg at CLion er en kommersiell IDE, og krever en lisens for full funksjonalitet eller om man tjener penger på det som utvikles (les deres lisensvilkår for mer informasjon). Det er imidlertid gratis for studenter og lærere gjennom JetBrains' utdanningsprogram, så det kan være et godt alternativ for de som kvalifiserer for det. CLion kan være overkill for enklere prosjekter eller for de som er nye til programmering, men det kan være et utmerket valg for mer komplekse prosjekter eller for de som ønsker å lære mer om avanserte C++ funksjoner og verktøy. Den kan også bidra med å gi en god innføring i kode struktur og organisering siden den har en del verktøy for å følge kode stil og organisering. Det er også verdt å merke seg at CLion har en del innebygde verktøy for testing og debugging, noe som kan være veldig nyttig for mer komplekse prosjekter. Det kan også være et godt valg for de som ønsker å lære mer om profesjonell programvareutvikling og verktøy som brukes i industrien.

## Oppsummering
Valg av editor eller IDE avhenger av dine personlige preferanser, erfaring, og behov. For nybegynnere kan Arduino IDE være et godt valg for å komme i gang med mikrokontrollerprogrammering, mens Visual Studio Code og CLion tilbyr mer avanserte funksjoner for de som ønsker å lære mer og jobbe med mer komplekse prosjekter. Det anbefales å prøve ut forskjellige editorer og IDE-er for å finne den som passer best for deg og dine behov i BIELSYS-studiet. Husk at det viktigste er å finne et verktøy som du er komfortabel med og som støtter dine læringsmål og prosjekter på en effektiv måte. Uansett hvilken editor eller IDE du velger, er det viktig å investere tid i å lære hvordan du bruker den effektivt, da dette kan ha en stor innvirkning på produktiviteten og læringsopplevelsen din gjennom BIELSYS-årene.

Forfaterens anbefaling er å bruke Visual Studio Code for de fleste studenter ved BIELSYS (men det er ikke garantert at stud asser kan hjelpe og det blir ikke undervist i det), da det gir en god balanse mellom brukervennlighet og avanserte funksjoner.

## Kjapt om bruk av KI/AI i editorer
Mange moderne editorer og IDE-er har nå integrert støtte for kunstig intelligens (KI) og maskinlæring (ML) for å forbedre kodefullføring, feilsøking, og generell utviklingsopplevelse. For eksempel, Visual Studio Code har utvidelser som GitHub Copilot, som bruker KI for å foreslå kode og hjelpe med autokomplettering basert på konteksten av det du skriver. CLion har også støtte for KI-baserte verktøy for kodeanalyse og refakturering. Disse verktøyene kan i noen tilfeller være til hjelp, men det er en del debatt rundt hvor nyttige de faktisk er. Blant annet så var det en undersøkelse som ble publisert i 2025 som indikerer at KI (med blant annet GPT-4o og Claude 3.5) ikke klarer å fikse mere en noen enkle oppgaver (https://futurism.com/openai-researchers-coding-fail). Det er også verdt å merke seg at KI-verktøy kan ha begrensninger og kan ikke alltid forstå konteksten eller intensjonen bak koden din, noe som kan føre til feil eller ineffektive forslag. Det kan også bidra med redusert forståelse av "din" kode, om du allikevel velger å bruke KI så er det noen anbefalinger som burde vurderes.

Vurder å skru av "inline suggestion": "Inline suggestion" gir deg bare et forslag til hva du skal gjøre videre, den har ikke noe mere kontekst en det du har skrevet så langt. I en del tilfeller viser det seg at den tar feil, noe som kan ta mere tid å rette opp i en det du hadde brukt på å løse problemet med andre metoder. Det er også mye enklere å miste oversikt over hva koden faktisk gjør i motsetning til hvordan den opererer i en chat. I en chat så vil den i det minste prøve å forklare hva den har gjort og "tenkt". Chat kan altså være et bedre alternativ en inline suggestions, men tilbyr fremdeles noen utfordringer, som nevnt så har KI en del problemer med komplekse oppgaver. Det er også noen miljømessige hensyn som bør gjøres før man velger å bruke KI, men det skal vi ikke se på her (video om KI resursbruk: https://youtu.be/H_c6MWk7PQc?si=s-Rn35bYn43XBtoY).
